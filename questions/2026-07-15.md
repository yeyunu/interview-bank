## 题目：Agent 的效果如何评估？

- 分层评估：单步工具调用准确率 → 轨迹级（步骤是否冗余/循环）→ 任务级端到端成功率
- 离线：固定任务集 + 可验证断言（类似 promptfoo/SWE-bench 思路）；在线：人工抽检 + 用户放弃率
- 关键坑：只看最终成功率会掩盖"绕远路"和"运气好"，要看成本（token/步数）与稳定性（多次运行方差）

## 题目：你的 Agent 项目和 Claude Code / Codex 这类主流 Agent 的主要差异是什么？

- 从定位差异说起：通用编码 Agent vs 我的场景化编排（举 mindcodex 的 orchestrator 实验为例）
- Harness 层差异：权限模型、上下文管理策略（压缩/检索）、工具集的收敛程度
- 诚实承认工程成熟度差距，突出自己在特定链路上做过的取舍和实验数据

## 题目：在 Agent 架构 Harness 层面，你做了哪些设计取舍？

- 上下文预算：什么进 system prompt、什么按需检索（代码索引 vs 全文塞入）
- 失败恢复：重试策略、turn 中断后的会话恢复（rollout 持久化）
- 安全边界：哪些工具免确认、哪些强制人工审批
